03 Loop Engine：Agent 的动力源泉

“用户发了一条指令，模型在一轮响应里究竟调用了多少次 API，工具是怎么排队执行的，什么时候才算真的「结束」？”

一轮 while(true) 的剖析

Loop Engine 是 Agent 的核心推进器，负责调度模型推理、实时解析工具调用流，并处理复杂的异常恢复。

工具调度逻辑

并发安全判定： CC 会动态分类工具副作用。Read/Search 类工具被视为并发安全，全速推进；而 Bash/Write 则被视为独占操作，必须强制串行。

Bash 级联中止 (Cascading Abort)： CC 定义了一个特殊的保护机制。一旦 Bash 串行任务失败，系统会通过 siblingAbortController.abort() 立即杀掉所有正在进行的并行读取任务。因为在有状态的 Terminal 中，前置命令失败通常意味着后续环境已失效。

容错恢复优先级链

当 API 返回 stop_reason 时，并不总是意味着 Loop 结束。系统会按序尝试恢复：

  • P0: max_output_tokens 溢出 —— 临时提升 max_tokens 到 64k 并注入“Continue”指令。
  • P1: prompt_too_long 溢出 —— 触发 Context Collapse 压缩算法，重新投递。
  • P2: Stop Hooks 触发 —— 调用用户自定义脚本验证结果，决定是否截断。
