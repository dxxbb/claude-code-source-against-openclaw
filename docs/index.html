AI Agent Runtime 架构深度解析

“在工业级场景下，Agent 的可靠性不只取决于模型，更取决于其运行时的设计模式。”

本项目通过对 Claude Code (CC) 与 OpenClaw (OC) 的源码级比对，系统性地梳理了 AI Agent 在实际运行过程中面临的核心挑战与解决方案。这不仅仅是一份源码审计，更是一场关于“本地化效率”与“分布式扩展”的设计哲学探讨。

专题目录

分析内容按 Agent 调度的生命周期分为以下五个维度，旨在揭示那些被 API 抽象所掩盖的技术细节：

  • 01 · Session (会话协议)

    剖析从用户意图到模型 Context 的映射链条。对比单通道 CLI 与多通道网关在会话新鲜度与路由策略上的差异。

  • 02 · Context (上下文治理)

    探讨如何通过动态/静态边界、Tombstone 机制以及自动化摘要策略，在保持长周期对话能力的同时优化 Prompt 缓存效率。

  • 03 · Loop Engine (执行引擎)

    深入 Loop 底层，拆解流式工具执行的并发控制、级联中止策略以及模型截断后的自动恢复机制。

  • 04 · Subagent (协作架构)

    解构任务分叉 (Fork) 与派生智能体的生命周期管理，理解如何通过父子谱系实现复杂的排查与执行流。

  • 05 · Memory (记忆系统)

    对比“无意识全量注入”与“检索式增强”两种记忆模型。理解 Session Memory 作为 Compaction 补偿机制的设计初衷。

设计哲学对照

通过对比你会发现，**Claude Code** 的设计核心是“极简与低延迟”，它通过深度耦合本地环境来榨取性能；而 **OpenClaw** 的设计核心是“解耦与扩展”，它通过标准化的接口和 Hook 机制来实现企业级的灵活性。

理解这两种截然不同的技术取向，将有助于你在构建自己的 Agent 系统时，做出更符合业务场景的决策。
